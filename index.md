# TTD vs TTA 비교 (초안)

이 페이지는 TTD와 TTA의 차이를 정리한 메모입니다.  
(추후 실험 결과/그림/표를 추가할 예정)

## 핵심 요약
- **TTA (Test-Time Adaptation)**: 테스트 단계에서 모델을 *직접* 업데이트해 분포 변화에 적응
- **TTD (Test-Time/Train-Time Defense or Deployment-oriented Robustness)**: 테스트 단계 업데이트에 의존하지 않고, 학습/설계/운영 관점에서 변화에 견디게 설계

## 왜 중요한가
배포 환경에서는 조명·센서·날씨·도메인 등으로 학습 분포와 테스트 분포가 달라져 성능이 급락할 수 있음.

## 장단점 (한눈에)
### TTA 장점
- 배포 후에도 환경 변화에 맞춰 적응 가능

### TTA 단점
- 온라인 업데이트 비용(시간/연산)
- 잘못 적응하면 성능 붕괴 위험(드리프트)
- 운영/안전/재현성 측면에서 부담

### TTD 장점
- 테스트 단계 업데이트가 없거나 적어 운영이 안정적
- 실시간 시스템에서 부담이 작음

### TTD 단점
- 예상 못한 변화에 대한 즉각 적응은 TTA보다 약할 수 있음

## 왜 이 글에서는 TTD 관점이 더 중요하다고 보는가

본인이 관심 있는 문제는 자율주행/실시간 인지와 같이  
**테스트 단계에서 모델 파라미터를 빈번하게 업데이트하기 어려운 환경**이다.

이러한 환경에서는  
- 예측 결과의 **재현성**
- 업데이트 실패 시의 **안전 리스크**
- 디버깅 가능성

이 매우 중요해지며,  
이 때문에 테스트 단계 적응에 강하게 의존하는 TTA보다는  
**배포 환경을 고려한 견고성 설계(TTD 관점)**가 더 현실적인 접근이라고 판단한다.

물론 TTA는 연구적으로 매우 강력한 도구이며,  
TTD 역시 TTA 계열 기법에서 얻은 통찰을 부분적으로 활용할 수 있다고 본다.
